# Tarea III: Investigación sobre el Uso y Aplicación de GitHub

## 1. Historia y Evolución de GitHub
GitHub fue fundado en 2008 por Tom Preston-Werner, Chris Wanstrath y PJ Hyett. Surgió como una plataforma para alojar proyectos utilizando el sistema de control de versiones Git. En 2018 fue adquirido por Microsoft, lo que impulsó su crecimiento hasta convertirlo en la comunidad de desarrolladores más grande del mundo. Ha evolucionado de ser un simple repositorio a una plataforma integral de DevOps con herramientas de integración y despliegue continuo.



## 2. Diferencias: Git vs. GitHub vs. GitLab
* **Git:** Es la herramienta de software local que gestiona el historial de cambios y versiones de un proyecto. Funciona sin necesidad de internet.
* **GitHub:** Plataforma en la nube que aloja repositorios de Git. Destaca por su enfoque en la colaboración social y el desarrollo de proyectos de código abierto.
* **GitLab:** Alternativa a GitHub que ofrece herramientas integradas de CI/CD más robustas de forma nativa y permite instalaciones en servidores privados (On-premise).

## 3. Conceptos Clave
* **Repositorio:** Espacio o carpeta del proyecto que contiene todos los archivos y el historial completo de cambios.
* **Commit:** Operación que captura y guarda los cambios realizados en el código en un momento específico.
* **Branch (Rama):** Línea de tiempo paralela que permite trabajar en nuevas funcionalidades sin afectar el código principal.
* **Merge:** Proceso de fusionar o unir los cambios de una rama secundaria a la rama principal.
* **Pull Request (PR):** Petición enviada al dueño del proyecto para que revise y apruebe la integración de nuevos cambios.
* **Fork:** Copia exacta de un repositorio ajeno en la cuenta propia del usuario para realizar modificaciones de forma independiente.
* **Issue:** Herramienta para reportar errores, sugerir mejoras o discutir tareas dentro del proyecto.



## 4. Funcionalidades Principales
* **Control de Versiones:** Capacidad de navegar por el historial del proyecto y revertir el código a estados anteriores si es necesario.
* **Colaboración:** Permite que múltiples desarrolladores trabajen sobre los mismos archivos simultáneamente mediante la gestión de conflictos.
* **GitHub Actions:** Sistema para la automatización de flujos de trabajo, como pruebas automáticas y despliegue de software.
* **GitHub Pages:** Servicio que permite el alojamiento gratuito de sitios web estáticos directamente desde un repositorio.

## 5. Guía: Crear un Repositorio Personal
1. Iniciar sesión en el portal oficial **GitHub.com**.
2. Hacer clic en el icono **"+"** ubicado en la parte superior derecha y seleccionar **"New repository"**.
3. Asignar un **Nombre** al proyecto (ej. `MiProyecto`).
4. Definir la visibilidad del repositorio como **Public** o **Private**.
5. Marcar la casilla **"Add a README file"** para inicializar el repositorio con un archivo de descripción.
6. Finalizar haciendo clic en el botón **"Create repository"**.
